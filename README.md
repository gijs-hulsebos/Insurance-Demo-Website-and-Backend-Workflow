### 🚀 Featured: [AI Insurance Claim Portal](https://insurance-demo-website-and-backend.vercel.app/)
This repository contains the frontend and API-bridge for an **Enterprise-Grade AI Insurance Triage System**. It serves as the interaction layer for a complex 10-phase n8n backend workflow.

**Key Architecture Features:**
* **Triage-Execution Layer:** Separates the user interface from the heavy lifting in the backend.
* **GDPR Privacy Logic:** Automatically anonymizes PII (Sensitive Data) before processing it through LLMs.
* **Human-in-the-Loop:** Integrated verification gates for manual review of high-stakes claim decisions.
* **Multi-Model Intelligence:** Uses agentic reasoning via Gemini and GPT-4o to analyze claim validity.

---

### 🛠️ Technical Stack (this project)
* **Frontend:** Next.js 15 (App Router), TypeScript, Tailwind CSS.
* **Backend Orchestration:** [n8n Agentic Workflow](https://github.com/gijs-hulsebos/n8n-automation/tree/main/Insurance-Demo).
* **API Layer:** Edge-ready API Routes for n8n Webhook communication.
* **Integrations:** Google Sheets (Vault/Logging), Gmail API, Google Workspace.

---

### 📊 Workflow Structure (10 Phases)
The frontend triggers a sophisticated backend process visualized in the accompanying documentation:
1.  **Phase 1-3:** Webhook intake, data structuring, and initial Google Sheets logging.
2.  **Phase 4-5:** PII Separation and AI Agent Triage (Gemini/GPT).
3.  **Phase 6-7:** Data re-merging and Coordinate Response logic.
4.  **Phase 8-10:** Manual Review (HITL), Status Updates, and Client Notification.
